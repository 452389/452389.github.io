web89

intval()函数：该函数是用于获取变量的整数值，又因其返回值 成功时返回 var 的 integer 值，失败时返回 0。 空的 array 返回 0，非空的 array 返回 1。//array是数组
实例
<?php
echo intval(42); // 42
echo intval(4.2); // 4
echo intval(‘42’); // 42
echo intval(‘+42’); // 42
echo intval(‘-42’); // -42
echo intval(042); // 34
echo intval(‘042’); // 42
echo intval(1e10); // 1410065408
echo intval(‘1e10’); // 1
echo intval(0x1A); // 26
echo intval(42000000); // 42000000
echo intval(420000000000000000000); // 0
echo intval(‘420000000000000000000’); // 2147483647
echo intval(42, 8); // 42
echo intval(‘42’, 8); // 34
echo intval(array()); // 0
echo intval(array(‘foo’, ‘bar’)); // 1
另外：intval处理开头是数字的字符串时，返回值为开头的数
且第二个参数设置0的时候，只要读到字符就会停止，不管他是不是科学计数法
第二个参数没设置的时候就不一样,他会正规的按照科学计数法
正则表达式匹配用的还不少，需学学
所以只要num是一个有值的数组
?num[]=1就可以达到要求

web91

$a=$_GET[‘cmd’];
if(preg_match(‘/^php$/im’, $a)){ //需满足这个匹配
if(preg_match(‘/^php$/i’, $a)){ //并且不能满足这个匹配
echo ‘hacker’;
}
else{
echo $flag;
}
}
else{
echo ‘nonononono’;
}

解读代码，比较第一个正则匹配与第二个正则匹配的不同，发现第一个里面的i后面多了一个m，m表示多行模式，多行模式的意思是每一行都进行正则匹配
所以本题的第一个正则是要求分行后至少有一行是字符串”php”，第二个正则只能匹配字符串”php”
因/m是换行匹配，所以第一个匹配是先换行再进行匹配，即相当于对传递参数的内容中的第二行进行匹配。而第二个正则匹配无换行匹配
所以用%0aphp(%0a是换行)
即可实现：第一次匹配的时候换行匹配到了php，第二次匹配因为没有进行换行匹配，所以就能成功绕过
%0a是\n的url编码，这里是不能用\n的

该题只要不满足第一个比较，满足第二个比较即可
可以将4476进行8进制或者十六进制编码即可 即：?num=010574或?num=0x117c，又或?num=4476e10
o,Q为八进制前缀，0x为16进制前缀
一些编程语言中常常以数字0开始表明该数字是八进制
php弱类型比较(==或!=)：php弱类型变量是指,在声明一个变量时,我们不需要指明它保存的数据类型。
即是：不需要比较的两边数据类型相同，在进行比较的时候，会先将字符串类型转化成相同，再比较。
在字符串与数字弱比较时，字符串会被转换成数值进行比较，字符串的开始部分决定了它的值，如果该字符串以合法的数值开始，则使用该数值，否则其值为0。
