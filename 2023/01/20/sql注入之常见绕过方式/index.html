sql注入之常见绕过方式

发表于 2023-01-20

双写绕过

如当我们进行联合查询时输入union select发现被过滤了
那我们便可以输入ununionion selselectect来进行过滤
有的时候会被过滤两次或更多这时我们可以也多输入几次来进行绕过

大小写绕过

如当我们进行联合查询时输入union select发现被过滤了
我们可以将所输入内容其中任何一个字母来进行大小写替换如：uNion select来进行过滤

空格绕过

如果空格被过滤掉了，那么我们可以用/**/，%09，%0a，%0d，%0b，（），+，-，tab键，两个空格，反引号等进行替换来达到绕过的目的

等号绕过

当等号被过滤掉时我们可以用like，<>来进行代替
<>等价于!=所以我们可以在<>前加个!就变成等于了

宽字节注入

前提条件

在实际应用中，为防止注入漏洞通常会把单引号处理下比如：
$id=$_GET[‘id’]；
select * from users where id = $id’
上边呢我们输入进去的 SQL 语句，都会赋值给 $id这个变量，然后会被包含在引号里，在PHP中引号里的内容是属于字符串的，所以我们直接在这里输入的一些 SQL 命令都会被当作字符串处理，这些命令就不会起到作用。
所以我们在字符型注入中才会先用单引号闭合下，再输入SQL命令，这样后面的命令才会执行。如果程序中在我们输入的单引号前加上个反斜杠比如：\’。这样的话这个单引号就是个字符了，没有单引号闭合的作用，所以就起到了防护的作用
为了突破这个防护，我们可以使用宽字节注入的方式进行突破：
我们的使用方式是：%df’
当我们在单引号前加上%df的时候，也就是输入%df’这样传入到数据库中就会变成：%df%5c’
在GBK的编码表中https://www.qqxiuzi.cn/zh/hanzi-gbk-bianma.php便会有相对应的字
另外%DE%5c,%E0%5c等都是可以的

注入步骤

一，在id=1后面添加单引号发现被转义
二，使用%df进行闭合:id=1%df’ and 1=1%23
